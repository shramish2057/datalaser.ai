## Overview

The Insights API triggers DataLaser's AI engine to analyze your data and surface actionable findings. Insights are generated at the project level, combining signals from all sources within a project. The anomaly detection endpoint returns statistical outliers identified during profiling and analysis.

---

## Generate Insights

Trigger insight generation for an entire project. DataLaser analyzes all sources in the project, cross-references patterns, and produces prioritized insights.

```
POST /api/insights/generate
```

### Request

<ParamField header="Authorization" type="string" required>
  Bearer token.
</ParamField>

<ParamField header="X-Project-Id" type="string" required>
  The project UUID.
</ParamField>

<ParamField body="project_id" type="string" required>
  The project UUID.
</ParamField>

<ParamField body="source_ids" type="array">
  Array of source UUIDs to include. Defaults to all sources in the project.
</ParamField>

<ParamField body="categories" type="array">
  Filter insight categories. Valid values: `trend`, `anomaly`, `correlation`, `distribution`, `quality`, `recommendation`.

  Defaults to all categories.
</ParamField>

<ParamField body="min_confidence" type="number">
  Minimum confidence score (0.0 to 1.0) for returned insights. Defaults to `0.7`.
</ParamField>

<ParamField body="max_insights" type="integer">
  Maximum number of insights to return. Defaults to `50`.
</ParamField>

### Example

```bash
curl -X POST https://datalaser.de/api/insights/generate \
  -H "Authorization: Bearer $DATALASER_API_KEY" \
  -H "X-Project-Id: proj_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "proj_abc123",
    "categories": ["trend", "anomaly", "correlation"],
    "min_confidence": 0.8,
    "max_insights": 20
  }'
```

### Errors

| Status | Code                  | Description                                   |
|--------|-----------------------|-----------------------------------------------|
| `404`  | `PROJECT_NOT_FOUND`   | Project ID does not exist.                    |
| `422`  | `NO_SOURCES`          | The project has no data sources to analyze.   |
| `422`  | `GENERATION_ERROR`    | Insight generation failed.                    |
| `408`  | `TIMEOUT`             | Generation exceeded the 120s timeout.         |

## Get Anomalies

Retrieve anomaly detection results for a project or specific source. Anomalies are detected automatically during profiling and insight generation.

```
GET /api/anomalies
```

### Query Parameters

<ParamField query="project_id" type="string" required>
  The project UUID.
</ParamField>

<ParamField query="source_id" type="string">
  Filter anomalies to a specific source. Returns all sources if omitted.
</ParamField>

<ParamField query="severity" type="string">
  Filter by severity. One of: `critical`, `warning`, `info`. Returns all severities if omitted.
</ParamField>

<ParamField query="column" type="string">
  Filter anomalies to a specific column name.
</ParamField>

<ParamField query="limit" type="integer">
  Maximum results to return. Defaults to `100`.
</ParamField>

<ParamField query="offset" type="integer">
  Pagination offset. Defaults to `0`.
</ParamField>

### Example


### Errors

| `404`  | `PROJECT_NOT_FOUND`   | Project ID does not exist.         |
