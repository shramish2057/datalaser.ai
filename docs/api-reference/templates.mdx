## Overview

Templates are pre-built analysis workflows that match specific data patterns. DataLaser ships with templates for common scenarios (churn analysis, revenue trends, cohort analysis, etc.) and automatically recommends applicable ones based on your column profiles. The AutoAnalyzer runs all 17 built-in analyses in a single call.

---

## Get Applicable Templates

Given column profiles, return all matching templates ranked by relevance.

```
POST /api/pipeline/templates
```

### Request

<ParamField body="action" type="string" required>
  Must be `"applicable"`.
</ParamField>

<ParamField body="source_id" type="string" required>
  The data source UUID.
</ParamField>

<ParamField body="column_profiles" type="array" required>
  Array of column profile objects. Typically obtained from the [Profile File](/api-reference/pipeline#profile-file) or [Profile Database](/api-reference/pipeline#profile-database) endpoints.

  Each object should include `name`, `inferred_type`, `null_pct`, `unique_count`, and statistics relevant to the column type.
</ParamField>

### Example

```bash
curl -X POST https://datalaser.de/api/pipeline/templates \
  -H "Authorization: Bearer $DATALASER_API_KEY" \
  -H "X-Project-Id: proj_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "applicable",
    "source_id": "src_8f3a1b2c",
    "column_profiles": [
      {
        "name": "revenue",
        "inferred_type": "float64",

```

### Errors

| Status | Code                  | Description                              |
|--------|-----------------------|------------------------------------------|
| `404`  | `SOURCE_NOT_FOUND`    | Source ID does not exist.                |
| `422`  | `INVALID_PROFILES`    | Column profiles are malformed.           |

## Run Template

Execute a specific analysis template against a data source. Returns structured findings, statistical results, and chart specifications.

```
POST /api/pipeline/templates
```

### Request

<ParamField body="action" type="string" required>
  Must be `"run"`.
</ParamField>

<ParamField body="template_id" type="string" required>
  The template identifier, e.g. `tpl_revenue_trend`.
</ParamField>

<ParamField body="source_id" type="string" required>
  The data source UUID.
</ParamField>

<ParamField body="column_mapping" type="object" required>
  Maps template-expected columns to actual column names in the source.
</ParamField>

<ParamField body="parameters" type="object">
  Optional template-specific parameters (e.g. date range, grouping level).
</ParamField>

| `422`  | `MAPPING_ERROR`       | Column mapping is invalid or incomplete.         |
| `422`  | `EXECUTION_ERROR`     | Template execution failed.                       |

## Run AutoAnalysis

Execute all 17 built-in analyses in a single call. The AutoAnalyzer profiles the data, selects applicable templates, runs them in parallel, and returns consolidated results.

```
POST /api/pipeline/auto-analysis
```

### Request

<ParamField body="source_id" type="string" required>
  The data source UUID.
</ParamField>

<ParamField body="include_charts" type="boolean">
  Include chart specifications in the response. Defaults to `true`.
</ParamField>

<ParamField body="max_analyses" type="integer">
  Maximum number of analyses to run. Defaults to `17` (all available).
</ParamField>

### Response

```json
{
  "source_id": "src_8f3a1b2c",
  "analyses_run": 11,
  "analyses_skipped": 6,
  "total_execution_time_seconds": 42.7,
  "results": [
    {
      "template_id": "tpl_revenue_trend",
      "template_name": "Revenue Trend Analysis",
      "status": "completed",
      "execution_time_seconds": 8.4,
      "findings_count": 3,
      "findings": [ "..." ],
      "charts": [ "..." ]
    },
    {
      "template_id": "tpl_descriptive_stats",
      "template_name": "Descriptive Statistics",
      "status": "completed",
      "execution_time_seconds": 2.1,
      "findings_count": 5,
      "findings": [ "..." ],
      "charts": [ "..." ]
    }
  ],
  "summary": {
    "total_findings": 28,
    "critical_findings": 2,

    "info_findings": 21
  },
  "skipped_reasons": [
    {
      "template_id": "tpl_geospatial",
      "reason": "No latitude/longitude columns detected."
    }
  ]
}
```

### Example

```bash
curl -X POST https://datalaser.de/api/pipeline/auto-analysis \
  -H "Authorization: Bearer $DATALASER_API_KEY" \
  -H "X-Project-Id: proj_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "source_id": "src_8f3a1b2c",
    "include_charts": true
  }'
```

<Note>
  AutoAnalysis can take 30-60 seconds for large datasets. The endpoint returns results synchronously. For very large sources, consider running individual templates instead.
</Note>

### Errors

| Status | Code                | Description                          |
|--------|---------------------|--------------------------------------|
| `404`  | `SOURCE_NOT_FOUND`  | Source ID does not exist.            |
| `408`  | `TIMEOUT`           | Analysis exceeded the 120s timeout.  |
| `422`  | `ANALYSIS_ERROR`    | One or more analyses failed.         |
