Every source in DataLaser is automatically profiled when first connected or synced. The **Data Health** page gives you a single quality score and a detailed column-by-column breakdown so you know exactly where issues live before you start analyzing.

## Quality Score

DataLaser assigns each source a score from **0 to 100** based on four weighted dimensions:

| Dimension | Weight | What It Measures |
|-----------|--------|-----------------|
| Completeness | 35% | Percentage of non-null values across all columns |
| Consistency | 25% | Type uniformity, format regularity, and encoding correctness |
| Uniqueness | 20% | Duplicate row rate and primary-key cardinality |
| Validity | 20% | Values within expected ranges, pattern compliance, referential integrity |

<Tip>
A score above **80** generally means your data is analysis-ready. Between **50-80** you should review suggestions in [Data Prep](/guides/data-prep). Below **50**, cleaning is strongly recommended before drawing conclusions.
</Tip>

The score is displayed as a colored ring on the source card:

- **80-100**: Green. Ready to analyze.
- **50-79**: Amber. Review recommended.
- **0-49**: Red. Cleaning needed.

## Column Overview Table

The health page includes a sortable table with one row per column:

| Field | Description |
|-------|-------------|
| **Column Name** | Original column header |
| **Detected Type** | Inferred data type (string, integer, float, date, boolean, categorical) |
| **Non-Null %** | Percentage of rows with a value |
| **Distinct Count** | Number of unique values |
| **Issues** | Count of warnings and characteristics flagged |
| **Quality** | Per-column mini-score (0-100) |

Click any column row to expand a detail panel with histograms, top values, and detected patterns.

## Issue Types

DataLaser classifies findings into three severity levels:

<AccordionGroup>
  <Accordion title="Warnings" icon="triangle-exclamation">
    Problems that are likely to distort analysis results if left unaddressed.

    - **High null rate**: More than 30% of values are missing.
    - **Type mismatch**: Column contains mixed types (e.g., numbers stored as strings).
    - **Outlier concentration**: Extreme values exceed 3 standard deviations and represent more than 5% of rows.
    - **Duplicate rows**: Identical rows detected beyond a threshold.
    - **Encoding errors**: Mojibake or unrecognized character sequences.
  </Accordion>
  <Accordion title="Characteristics" icon="circle-info">
    Observations that are not necessarily problems but worth knowing about.

    - **High cardinality**: Column has many unique values relative to row count (e.g., free-text fields).
    - **Skewed distribution**: Values concentrate heavily on one side.
    - **Sparse column**: Fewer than 10% of rows have a value (may be optional field).
    - **Constant column**: Every row has the same value.
    - **Date gaps**: Missing dates in an otherwise continuous time series.
  </Accordion>
  <Accordion title="Info" icon="lightbulb">
    Neutral metadata surfaced for context.

    - **Detected language**: German or English column names and values identified.
    - **Inferred format**: Date patterns, currency symbols, or phone formats detected.
    - **Sample values**: Representative values shown for quick inspection.
  </Accordion>
</AccordionGroup>

## Warnings vs. Characteristics

| | Warnings | Characteristics |
|---|----------|----------------|
| **Impact** | Can distort analysis results | Informational; may or may not need action |
| **Action** | Fix before analyzing | Review and decide |
| **Shown as** | Red badge | Blue badge |
